## Install in one command

```bash
git clone https://github.com/tai-framework/tai.git ~/tai && ~/tai/setup
```

Symlinks all commands, agents, and skills to `~/.claude/`. No binaries, no dependencies — pure markdown.

## Three tiers for every scope of work

| Tier | Command | Scope | Outcome |
|------|---------|-------|---------|
| **Task** | `/tai-task` | Minutes, 1–3 files | Single commit, no PR |
| **Feature** | `/tai-feature` | Hours, 3–10 files | Agent Team, PR |
| **Mission** | `/tai-mission` | Days/weeks, 10+ files | Multiple features, multiple PRs |

### Pick the right tier

- Fix a bug, change a color, rename something → **Task**
- Add a feature, build a flow, connect systems → **Feature**
- Build an entire system from scratch → **Mission**

When in doubt, start smaller. A task can always be promoted to a feature.

## Quality pipeline — no opt-out

Every tier runs this after implementation:

```
1. pnpm lint    (always)
2. pnpm build   (always)
3. pnpm test    (always)
4. browser      (smart detect)

Stop on first failure. Never commit broken code.
```

## What's included

- **23 slash commands** — task, feature, mission, debug, refactor, review, commit, ship, and more
- **2 global agents** — `tai-explorer` (read-only context) and `tai-implementer` (fallback implementation)
- **8 skills** — research, audit, test generation, changelog, dogfood testing, and more
- **5 hooks** — quality gate, branch guard, destructive command protection

## Optional CLI

```bash
cd ~/tai/cli && cargo install --path .
```

`tai doctor`, `tai list`, `tai add` — power tools for managing your setup.
